# 📘 フェイスカリング（Face Culling）

> **目標：** ワインディングオーダーとフェイスカリングの仕組みを理解し、不要な裏面の描画を省略してパフォーマンスを向上させる方法を習得する。

---

## 📖 フェイスカリングとは

3Dオブジェクトを描画する際、カメラに対して**裏側を向いている面（バックフェイス）**は通常見えません。閉じたメッシュ（立方体、球体など）では、裏面は必ず他の面で隠されています。

**フェイスカリング** は、こうした見えない面の描画を**スキップ**する最適化技術です。

```
立方体をカメラの前に置いた場合：

      ┌──────────┐
     ╱          ╱│
    ╱  上面    ╱ │
   ┌──────────┐  │
   │          │  │  ← 右面（一部見える）
   │  前面    │  ┘
   │ (見える) │ ╱   ← 底面（見えない＝バックフェイス）
   │          │╱    ← 背面（見えない＝バックフェイス）
   └──────────┘     ← 左面（見えない＝バックフェイス）

カメラから見える面: 3面（前面、上面、右面）
見えない面:        3面（背面、底面、左面）

→ カリングで描画量を約半分に削減！
```

---

## 📖 ワインディングオーダー（巻き方向）

OpenGLはどの面が「表」でどの面が「裏」かを、**頂点の定義順序（ワインディングオーダー）**で判断します。

### 反時計回り（Counter-Clockwise: CCW）

```
     v1
     ╱╲
    ╱  ╲
   ╱ → ╲     反時計回り（CCW）
  ╱  表  ╲    = フロントフェイス（デフォルト）
 v3───────v2

 頂点の順序: v1 → v2 → v3（反時計回り）
```

### 時計回り（Clockwise: CW）

```
     v1
     ╱╲
    ╱  ╲
   ╱ ← ╲     時計回り（CW）
  ╱  裏  ╲    = バックフェイス（デフォルト）
 v2───────v3

 頂点の順序: v1 → v2 → v3（時計回り）
```

**デフォルトの規則：**
- **反時計回り（CCW）** = フロントフェイス（表面）
- **時計回り（CW）** = バックフェイス（裏面）

### なぜ「見る方向」で変わるのか

同じ三角形でも、表から見るか裏から見るかで、頂点の巻き順が **逆転** します。

```
表から見た場合:           裏から見た場合:
     v1                       v1
     ╱╲                       ╱╲
    ╱  ╲                     ╱  ╲
   ╱ CCW╲     ←─────→      ╱ CW ╲
  v3─────v2               v2─────v3

 反時計回り                  時計回り
 = フロントフェイス           = バックフェイス
 = 描画される               = カリングされる
```

---

## 📖 ワインディングオーダーの判定方法

OpenGLは、画面上に投影された2D座標での頂点順序から巻き方向を判定します。数学的には**外積（クロス積）**の符号を使います。

### 外積による判定

三角形の辺ベクトルの外積のZ成分の符号で判定します。

$$\vec{e_1} = v_2 - v_1$$
$$\vec{e_2} = v_3 - v_1$$
$$cross_z = e_{1x} \cdot e_{2y} - e_{1y} \cdot e_{2x}$$

| $cross_z$ の符号 | ワインディング |
|:-:|:-:|
| 正（> 0） | 反時計回り（CCW）= フロントフェイス |
| 負（< 0） | 時計回り（CW）= バックフェイス |
| 0 | 退化三角形（面積ゼロ） |

```
例: v1=(0,2), v2=(2,0), v3=(0,0)（スクリーン座標）

e1 = v2 - v1 = (2-0, 0-2)  = (2, -2)
e2 = v3 - v1 = (0-0, 0-2)  = (0, -2)

cross_z = 2*(-2) - (-2)*0 = -4 - 0 = -4

cross_z < 0 → CW（時計回り）→ バックフェイス
```

## 📖 フェイスカリングの有効化

### 基本設定

```cpp
// フェイスカリングを有効化
glEnable(GL_CULL_FACE);
```

これだけで、バックフェイス（裏面）が描画されなくなります。

### glCullFace — カリング対象の設定

```cpp
glCullFace(GLenum mode);
```

| モード | カリングされる面 |
|--------|----------------|
| `GL_BACK` | バックフェイス（裏面）。**デフォルト** |
| `GL_FRONT` | フロントフェイス（表面） |
| `GL_FRONT_AND_BACK` | 両面。三角形が一切描画されない（ワイヤフレーム用途等） |

```cpp
// デフォルト：裏面をカリング
glCullFace(GL_BACK);

// 表面をカリング（特殊用途：シャドウボリューム等）
glCullFace(GL_FRONT);
```

### glFrontFace — フロントフェイスの定義

```cpp
glFrontFace(GLenum mode);
```

| モード | フロントフェイスの定義 |
|--------|---------------------|
| `GL_CCW` | 反時計回り。**デフォルト** |
| `GL_CW` | 時計回り |

```cpp
// デフォルト：反時計回りが表面
glFrontFace(GL_CCW);

// 時計回りを表面に変更（モデルデータの都合に合わせる場合）
glFrontFace(GL_CW);
```

## 📖 カリングが適切でない場面

フェイスカリングは万能ではありません。以下の状況では無効にする必要があります。

### 1. 平面（板ポリゴン）

草や看板のように**表裏を見せたい**オブジェクトでは、裏面をカリングすると片面が消えます。

```
草のスプライト:

  表から見た場合:      裏から見た場合（カリングあり）:
  ┌──────┐            （何も見えない！）
  │ 🌿   │
  └──────┘

→ カリングを無効にするか、両面分のポリゴンを用意する
```

### 2. 半透明オブジェクト

半透明の壁やガラスでは、裏側の面も透けて見えます。

```cpp
// 半透明オブジェクトの描画時はカリングを一時無効化
glDisable(GL_CULL_FACE);
drawTransparentObject();
glEnable(GL_CULL_FACE);
```

### 3. 開いたメッシュ

カップの内側のように、メッシュが閉じていない場合は裏面も見る可能性があります。

```
閉じたメッシュ:          開いたメッシュ:
    ┌────┐                ┌────
    │    │                │    
    │    │                │  ← 内側が見える
    │    │                │    
    └────┘                └────
  裏面は見えない          裏面が見える可能性
  → カリングOK            → カリング注意
```

### 4. スケールの反転

`glm::scale` で負のスケール値を使うと、ワインディングオーダーが反転します。

```cpp
// 負のスケールでワインディングが反転する
model = glm::scale(model, glm::vec3(-1.0f, 1.0f, 1.0f));
// → フロントフェイスとバックフェイスが入れ替わる
// → glFrontFace(GL_CW) に切り替えるか、カリングを無効に
```

## 📖 実践的な設定パターン

### パターン1: 基本的なシーン描画

```cpp
glEnable(GL_CULL_FACE);

while (rendering)
{
    glEnable(GL_CULL_FACE);
    drawOpaqueObjects();     // 不透明な閉じたメッシュ

    glDisable(GL_CULL_FACE);
    drawGrass();             // 平面オブジェクト
    drawTransparentObjects(); // 半透明

    glEnable(GL_CULL_FACE);
}
```

### パターン2: シャドウマッピング（Peter Panning対策）

シャドウマップ生成時に表面をカリングし、裏面のみを描画することで、セルフシャドウのアーティファクト（Peter Panning）を軽減できます。

```cpp
// シャドウマップ描画パス
glCullFace(GL_FRONT);  // 表面をカリング（裏面を描画）
renderSceneToShadowMap();

// 通常描画パス
glCullFace(GL_BACK);   // 裏面をカリングに戻す
renderScene();
```

## 📖 カリングとGPUパイプライン

カリングはGPUパイプラインの**プリミティブアセンブリの後、ラスタライズの前**に実行されます。つまり、バックフェイスのフラグメントは一切生成されず、フラグメントシェーダも実行されません。これがパフォーマンス向上の理由です。

---

## 📖 gl_FrontFacing 変数

フラグメントシェーダの組み込み変数 `gl_FrontFacing` で表裏を判定できます。カリングを無効にして表と裏で異なるテクスチャを表示する際に便利です。

```glsl
#version 330 core
out vec4 FragColor;
in vec2 TexCoords;
uniform sampler2D frontTexture;
uniform sampler2D backTexture;

void main()
{
    if (gl_FrontFacing)
        FragColor = texture(frontTexture, TexCoords);
    else
        FragColor = texture(backTexture, TexCoords);
}
```

## 💡 ポイントまとめ

| 項目 | 内容 |
|------|------|
| フェイスカリング | 裏面を描画しないことでパフォーマンス向上 |
| ワインディングオーダー | 頂点の巻き順で表裏を判定 |
| デフォルト表面 | 反時計回り（CCW）がフロントフェイス |
| 有効化 | `glEnable(GL_CULL_FACE)` |
| カリング対象 | `glCullFace(GL_BACK)` がデフォルト |
| 表面の定義 | `glFrontFace(GL_CCW)` がデフォルト |
| 判定方法 | 外積のZ成分の符号で巻き順を判定 |
| 効果 | 閉じたメッシュの描画量を約50%削減 |
| 無効にすべき場面 | 平面、半透明、開いたメッシュ、負スケール |
| `gl_FrontFacing` | GLSL で表裏を判定する組み込み変数 |

## ✏️ ドリル問題

### 問題1: 穴埋め
フェイスカリングを有効化し、裏面をカリングする設定を完成させてください。

```cpp
glEnable(______);
glCullFace(______);
glFrontFace(______);
```

<details><summary>📝 解答</summary>

```cpp
glEnable(GL_CULL_FACE);
glCullFace(GL_BACK);
glFrontFace(GL_CCW);
```

`GL_BACK`（裏面をカリング）と `GL_CCW`（反時計回りが表面）はどちらもデフォルト値です。`glEnable(GL_CULL_FACE)` だけでも同じ動作になりますが、明示的に設定するのが良い習慣です。

</details>

---

### 問題2: 選択問題
三角形の頂点をスクリーン上で v1=(0,0), v2=(1,0), v3=(0,1) の順序で定義した場合、このワインディングオーダーは？

- (A) 反時計回り（CCW）
- (B) 時計回り（CW）
- (C) どちらでもない
- (D) 情報不足で判断できない

<details><summary>📝 解答</summary>

**(A) 反時計回り（CCW）**

外積のZ成分を計算します：
- $\vec{e_1} = v_2 - v_1 = (1, 0)$
- $\vec{e_2} = v_3 - v_1 = (0, 1)$
- $cross_z = 1 \times 1 - 0 \times 0 = 1 > 0$

正の値 → 反時計回り（CCW）→ フロントフェイスです。

```
 v3(0,1)
   │╲
   │  ╲   CCW（反時計回り）
   │    ╲
 v1(0,0)──v2(1,0)
```

</details>

---

### 問題3: 選択問題
フェイスカリングが **不適切** な場面はどれですか？（複数選択可）

- (A) 閉じた立方体の描画
- (B) 草のスプライト（板ポリゴン）の描画
- (C) 半透明なガラスの描画
- (D) 閉じた球体の描画

<details><summary>📝 解答</summary>

**(B) と (C)**

- **(B) 草のスプライト**: 板ポリゴンは表裏の両方から見えるため、裏面をカリングすると片方向から見えなくなります。
- **(C) 半透明なガラス**: 裏面も透けて見えるため、カリングすると不自然になります。

(A) と (D) は閉じたメッシュなので、裏面は他の面で必ず隠されており、カリングして問題ありません。

</details>

---

### 問題4: 計算問題
三角形の頂点がスクリーン座標で v1=(3,1), v2=(1,4), v3=(5,4) の場合、外積のZ成分を計算し、このフェイスはフロントかバックか答えてください。

<details><summary>📝 解答</summary>

$$\vec{e_1} = v_2 - v_1 = (1-3, 4-1) = (-2, 3)$$
$$\vec{e_2} = v_3 - v_1 = (5-3, 4-1) = (2, 3)$$
$$cross_z = (-2) \times 3 - 3 \times 2 = -6 - 6 = -12$$

$cross_z = -12 < 0$ → **時計回り（CW）** → **バックフェイス**

デフォルト設定（`glFrontFace(GL_CCW)`、`glCullFace(GL_BACK)`）ではこの三角形はカリングされます。

</details>

---

### 問題5: 穴埋め
シャドウマップ生成時に表面をカリングし、通常描画で裏面をカリングする設定を完成させてください。

```cpp
// シャドウマップ描画パス
glCullFace(______);
renderSceneToShadowMap();

// 通常描画パス
glCullFace(______);
renderScene();
```

<details><summary>📝 解答</summary>

```cpp
// シャドウマップ描画パス
glCullFace(GL_FRONT);
renderSceneToShadowMap();

// 通常描画パス
glCullFace(GL_BACK);
renderScene();
```

シャドウマップで `GL_FRONT`（表面をカリング）にすると、裏面の深度が記録されます。これにより、通常描画時のセルフシャドウアーティファクト（Peter Panning）を軽減できます。

</details>

---

### 問題6: 記述問題
`glm::scale(model, glm::vec3(-1.0f, 1.0f, 1.0f))` を適用すると、フェイスカリングの結果はどう変わりますか？その理由を説明してください。

<details><summary>📝 解答</summary>

X軸に -1 のスケールを適用すると、すべての頂点のX座標が反転します。これにより三角形の頂点が「鏡像反転」され、**ワインディングオーダーが逆転** します。

元々 CCW（フロント）だった面が CW（バック）になり、CW（バック）だった面が CCW（フロント）になります。

結果として、本来描画されるべき表面がカリングされ、本来カリングされるべき裏面が描画されます。対策は：
1. `glFrontFace(GL_CW)` に一時的に切り替える
2. カリングを一時無効化する

のいずれかです。奇数個の軸で負のスケールを適用するとこの現象が起きます。

</details>

## 🔨 実践課題

### 課題1: カリングの効果を確認 ⭐
フェイスカリングの有無でパフォーマンスと見た目がどう変わるか実験してください。

**チェックポイント:**
- [ ] 大量のキューブ（100個以上）をシーンに配置

- [ ] ワイヤフレームモードで裏面の三角形が消えていることを確認
- [ ] キー入力でカリングのON/OFFを切り替え
