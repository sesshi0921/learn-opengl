# 深度テスト（Depth Testing）

> **目標：** OpenGLの深度テストの仕組みを理解し、深度バッファの非線形性やZ-fighting問題に対処できるようになる。

---

## フレームバッファと深度バッファ

OpenGLでシーンを描画すると、結果は**フレームバッファ**に書き込まれます。フレームバッファは複数のバッファで構成されています。

```
┌─────────────────────────────────────────────┐
│ フレームバッファ │
│ │
│ ┌─────────────┐ ┌─────────────┐ │
│ │ カラーバッファ │ │ 深度バッファ │ │
│ │ (RGBA) │ │ (Z値) │ │
│ │ 各ピクセル │ │ 各ピクセル │ │
│ │ 32bit │ │ 24bit │ │
│ └─────────────┘ └─────────────┘ │
│ │
│ ┌──────────────────┐ │
│ │ ステンシルバッファ │ │
│ │ (マスク値) │ │
│ │ 各ピクセル 8bit │ │
│ └──────────────────┘ │
└─────────────────────────────────────────────┘
```

- **カラーバッファ**: 最終的な色（RGBA）を格納する
- **深度バッファ（Zバッファ）**: 各ピクセルの深度情報を格納する（通常24ビット）
- **ステンシルバッファ**: ピクセル単位のマスク情報（通常8ビット）

深度バッファはウィンドウシステムが自動的に作成します。深度値は `0.0`〜`1.0` の範囲で格納されます。

---

## 深度テストの仕組み

深度テストとは、「新しいフラグメントを描画すべきかどうか」を **深度値（Z値）の比較** で決定するプロセスです。

### 処理の流れ

```
フラグメントシェーダ出力
        │
        ▼
┌──────────────────────┐
│ フラグメントの深度値を取得 │
│ (gl_FragCoord.z) │
└──────────┬───────────┘
           │
           ▼
┌──────────────────────────────┐
│ 深度バッファの既存値と比較 │
│ (glDepthFunc で設定した関数) │
└──────────┬───────────────────┘
           │
     ┌─────┴─────┐
     │ │
  テスト通過 テスト不合格
     │ │
     ▼ ▼
 描画＆深度 フラグメント
 バッファ更新 破棄
```

**なぜ深度テストが必要なのか？**
深度テストがなければ、後から描画されたオブジェクトが常に手前に見えてしまいます。3Dシーンで「奥にあるものは手前のもので隠れる」という自然な見え方を実現するために不可欠です。

---

## 深度テストの有効化

深度テストはデフォルトでは **無効** です。明示的に有効化する必要があります。

```cpp
// 深度テストを有効化
glEnable(GL_DEPTH_TEST);
```

そして**毎フレーム**の描画ループの先頭で、深度バッファをクリアします。

```cpp
// レンダリングループ
while (!glfwWindowShouldClose(window))
{
    // カラーバッファと深度バッファを同時にクリア
    glClear(GL_COLOR_BUFFER_BIT | GL_DEPTH_BUFFER_BIT);

    // ... シーンの描画 ...

    glfwSwapBuffers(window);
    glfwPollEvents();
}
```

> ⚠️ `GL_DEPTH_BUFFER_BIT` のクリアを忘れると、前フレームの深度値が残り、描画がおかしくなります。

---

## 深度テスト関数

`glDepthFunc` で比較関数を設定します。デフォルトは `GL_LESS` です。

```cpp
glDepthFunc(GL_LESS); // デフォルト
```

### 関数一覧

| 関数 | 条件 | 意味 |
|------|------|------|
| `GL_ALWAYS` | 常に通過 | 深度テスト無効と同等 |
| `GL_NEVER` | 常に不合格 | 何も描画されない |
| `GL_LESS` | `fragment.z < buffer.z` | **デフォルト**。手前のものが優先 |
| `GL_LEQUAL` | `fragment.z <= buffer.z` | スカイボックス描画で使用（z=1.0で描画） |
| `GL_GREATER` | `fragment.z > buffer.z` | 奥のものが優先 |
| `GL_GEQUAL` | `fragment.z >= buffer.z` | Greater or Equal |
| `GL_EQUAL` | `fragment.z == buffer.z` | 完全一致のみ |
| `GL_NOTEQUAL` | `fragment.z != buffer.z` | 不一致のみ |

**使い分けの例：**
- `GL_LESS`：一般的な3Dシーン描画（デフォルト）
- `GL_LEQUAL`：スカイボックスを最奥（z=1.0）に描画する場合
- `GL_ALWAYS`：デバッグ用。すべてのフラグメントを上書き描画

## 深度値の非線形性

### 線形深度 vs 実際の深度

理想的には深度値は線形であるべきですが、OpenGLの深度バッファは **非線形** です。

```
実際の深度バッファの精度分布：

near far
 │◄──── 高精度 ────►│◄────── 低精度 ──────────►│
 │██████████████████│░░░░░░░░░░░░░░░░░░░░░░░░│
 0.0 1.0

「近くの精度は高い」が「遠くの精度は極端に低い」
```

**なぜ非線形なのか？**
透視投影では、近くのオブジェクトの方が細かい深度の差が重要です。遠くのオブジェクトは小さく見えるため、精度が低くても問題になりにくいという考え方です。

### 深度値の変換式

ビュー空間のZ値（線形）から深度バッファの値への変換：

$$
F_{depth} = \frac{\frac{1}{z} - \frac{1}{near}}{\frac{1}{far} - \frac{1}{near}}
$$

これは $1/z$ に比例する式です。この非線形性により：

| z（ビュー空間） | near=0.1, far=100 での深度バッファ値 |
|:-:|:-:|
| 0.1（near） | 0.0 |
| 1.0 | ≈ 0.9 |
| 10.0 | ≈ 0.99 |
| 50.0 | ≈ 0.998 |
| 100.0（far） | 1.0 |

**驚くべきこと：** z=1.0（nearの10倍）の時点で既に深度値の90%を使い切っています！

```
深度バッファ値の分布（near=0.1, far=100）:

z=0.1 z=1 z=10 z=50 z=100
 │ │ │ │ │
 ▼ ▼ ▼ ▼ ▼
 ├──────┼─────┼───────────┼──────────┤
 0.0 0.9 0.99 0.998 1.0
       ↑
   ここまでで精度の90%を消費！
```

## 線形化深度のGLSL実装

深度バッファの値を可視化するために、線形深度に変換する関数を作成します。

```glsl
// フラグメントシェーダ
#version 330 core
out vec4 FragColor;

float near = 0.1;
float far = 100.0;

// 非線形深度値を線形深度に変換
float LinearizeDepth(float depth)
{
    // まず [0,1] の深度値を NDC の [-1,1] に戻す
    float z = depth * 2.0 - 1.0;
    // 逆変換で線形のビュー空間Z値を取得
    return (2.0 * near * far) / (far + near - z * (far - near));
}

void main()
{
    // gl_FragCoord.z は [0,1] の非線形深度値
    float depth = LinearizeDepth(gl_FragCoord.z) / far;
    // 線形深度をグレースケールで可視化
    FragColor = vec4(vec3(depth), 1.0);
}
```

### 可視化結果の解釈

- **真っ黒（0.0に近い）**: カメラに非常に近い
- **グレー**: 中間距離
- **真っ白（1.0に近い）**: far平面に近い

非線形な深度値をそのまま表示すると、ほぼ全体が白く見えます。`LinearizeDepth` を使うことで、距離に応じたグラデーションが見えるようになります。

## Z-fighting（Zファイティング）

### Z-fightingとは？

2つのサーフェスがほぼ同じ深度値を持つとき、深度テストの結果がピクセルごとに不安定になり、**ちらつき模様** が発生します。これがZ-fightingです。

```
Z-fightingの見た目（2つの重なった平面）:

┌─────────────────┐
│█░█░░█░█░█░░█░█░│ ← ピクセルごとに
│░█░██░░█░░██░█░░│ どちらの面が
│█░░█░█░░█░█░░█░█│ 手前か変わる
│░██░░█░██░░█░██░│
└─────────────────┘
  モアレ状のちらつきパターン
```

### 3つの対策

#### 対策1: オブジェクトを近づけすぎない

最も簡単な方法は、重なるサーフェス間に微小な**オフセット**を与えることです。

```cpp
// 2つ目の床を少しだけ上に配置
model = glm::translate(model, glm::vec3(0.0f, 0.001f, 0.0f));
```

ただし手動調整が必要で、大規模シーンでは管理が困難です。

#### 対策2: near平面をできるだけ遠くに設定

near平面を大きくすると、有効な深度範囲での精度が上がります。

```cpp
// 悪い例：near が近すぎる
glm::mat4 proj = glm::perspective(glm::radians(45.0f),
    aspect, 0.001f, 1000.0f); // near=0.001 → 精度が極端に低い

// 良い例：near を適切に設定
glm::mat4 proj = glm::perspective(glm::radians(45.0f),
    aspect, 0.1f, 100.0f); // near=0.1 → 精度が大幅に改善
```

**経験則：** `near` は可能な限り大きく（0.1以上推奨）、`far` は必要最小限に。

#### 対策3: 高精度深度バッファを使用

デフォルトの24ビットから32ビットの深度バッファに変更できます。精度は向上しますが、メモリ消費も増えます。

## 深度バッファへの書き込み制御

`glDepthMask` を使うと、深度テストは有効のまま、深度バッファへの**書き込みだけ**を無効化できます。

```cpp
// 深度バッファへの書き込みを無効化
glDepthMask(GL_FALSE);

// この間の描画は深度テストは行うが、深度バッファを更新しない
// → 半透明オブジェクトの描画などに使用

// 書き込みを再度有効化
glDepthMask(GL_TRUE);
```

**使用場面：**
- 半透明オブジェクトの描画（深度値を更新すると後ろのオブジェクトが消える）
- パーティクルシステムの描画
- デカール（壁に貼り付けるテクスチャ）の描画

## 💡 ポイントまとめ

| 項目 | 内容 |
|------|------|
| 深度バッファ | 各ピクセルの深度値（0.0〜1.0）を格納。通常24ビット |
| 有効化 | `glEnable(GL_DEPTH_TEST)` |
| 毎フレームクリア | `glClear(GL_DEPTH_BUFFER_BIT)` を忘れずに |
| デフォルト関数 | `GL_LESS`（手前のフラグメントが勝つ） |
| 非線形性 | 深度値は $1/z$ に比例。near付近に精度が集中 |
| LinearizeDepth | `(2*near*far) / (far+near-z*(far-near))` |
| Z-fighting | 同一深度のサーフェスで発生するちらつき |
| Z-fighting対策 | ①オフセット ②nearを遠く ③高精度バッファ |
| 書き込み制御 | `glDepthMask(GL_FALSE)` で書き込みのみ無効化 |
| `GL_LEQUAL` | スカイボックス描画に使用（z=1.0でも通過） |

## ドリル問題

### 問題1: 穴埋め
深度テストを有効にし、毎フレーム深度バッファをクリアするコードを完成させてください。

```cpp
glEnable(______);

// レンダリングループ内
glClear(GL_COLOR_BUFFER_BIT | ______);
```

<details><summary> 解答</summary>

```cpp
glEnable(GL_DEPTH_TEST);

// レンダリングループ内
glClear(GL_COLOR_BUFFER_BIT | GL_DEPTH_BUFFER_BIT);
```

</details>

---

### 問題2: 選択問題
深度テスト関数のデフォルト値はどれですか？

- (A) `GL_ALWAYS`
- (B) `GL_LEQUAL`
- (C) `GL_LESS`
- (D) `GL_EQUAL`

<details><summary> 解答</summary>

**(C) `GL_LESS`**

フラグメントの深度値が既存のバッファ値**より小さい**場合に通過します。つまり、カメラに近いフラグメントが優先されます。

</details>

---

### 問題3: 計算問題
near=0.1、far=100 のとき、ビュー空間で z=1.0 にあるフラグメントの深度バッファ値を概算してください。

$$F_{depth} = \frac{\frac{1}{z} - \frac{1}{near}}{\frac{1}{far} - \frac{1}{near}}$$

<details><summary> 解答</summary>

$$F_{depth} = \frac{\frac{1}{1.0} - \frac{1}{0.1}}{\frac{1}{100} - \frac{1}{0.1}} = \frac{1.0 - 10.0}{0.01 - 10.0} = \frac{-9.0}{-9.99} \approx 0.901$$

z=1.0 の時点で深度バッファの約90%を既に使用しています。残りの z=1.0〜100.0 の範囲は、深度バッファの約10%しか使えません。

</details>

---

### 問題4: 選択問題
Z-fightingの対策として**適切でない**ものはどれですか？

- (A) near平面をカメラから離す
- (B) 深度バッファの精度を上げる（24bit→32bit）
- (C) far平面を極端に遠くする
- (D) 重なるサーフェスに微小なオフセットを入れる

<details><summary> 解答</summary>

**(C) far平面を極端に遠くする**

far平面を遠くすると、深度バッファがカバーする範囲が広がり、精度が**低下**します。Z-fightingは悪化します。対策はnear平面を遠くすること（far-near の比率を小さくすること）です。

</details>

---

### 問題5: 穴埋め
深度テストは行うが深度バッファへの書き込みを無効化するコードを書いてください。

```cpp
_______(GL_FALSE);
// 半透明オブジェクトの描画
drawTransparentObjects();
// 書き込みを再度有効化
_______(GL_TRUE);
```

<details><summary> 解答</summary>

```cpp
glDepthMask(GL_FALSE);
// 半透明オブジェクトの描画
drawTransparentObjects();
// 書き込みを再度有効化
glDepthMask(GL_TRUE);
```

`glDepthMask` はテスト自体を無効化するのではなく、**書き込みのみ**を制御します。

</details>

---

### 問題6: 穴埋め（GLSL）
以下の `LinearizeDepth` 関数の空欄を埋めてください。

```glsl
float LinearizeDepth(float depth)
{
    float z = depth * 2.0 - 1.0;
    return (2.0 * ______ * ______) / (______ + ______ - z * (______ - ______));
}
```

<details><summary> 解答</summary>

```glsl
float LinearizeDepth(float depth)
{
    float z = depth * 2.0 - 1.0;
    return (2.0 * near * far) / (far + near - z * (far - near));
}
```

この関数は NDC（正規化デバイス座標）の深度値を、線形なビュー空間のZ値に戻します。

</details>

---

### 問題7: 記述問題
スカイボックスの描画で `GL_LESS` ではなく `GL_LEQUAL` を使う理由を説明してください。

<details><summary> 解答</summary>

スカイボックスは最も奥（深度値 = 1.0）に描画されます。深度バッファのクリア値も 1.0 です。`GL_LESS` では `1.0 < 1.0` が `false` となりテストに失敗します。`GL_LEQUAL` なら `1.0 <= 1.0` が `true` となり、スカイボックスのフラグメントが描画されます。

</details>

## 実践課題

### 課題1: 深度バッファの可視化 
既存の3Dシーンに対して、深度バッファをグレースケールで可視化するシェーダを実装してください。

**チェックポイント:**
- [ ] `gl_FragCoord.z` をそのまま出力 → ほぼ白一色になることを確認
- [ ] `LinearizeDepth` で線形化して出力 → 距離に応じたグラデーションを確認
- [ ] near/far の値を変えて精度分布の変化を観察

---

### 課題2: 深度テスト関数の比較 
複数のキューブを異なる深度に配置し、`glDepthFunc` を切り替えることで描画結果がどう変わるか観察してください。

**チェックポイント:**
- [ ] `GL_LESS`（デフォルト）で正しい前後関係を確認
- [ ] `GL_GREATER` に切り替え → 結果の違いを説明できる
- [ ] `GL_ALWAYS` に切り替え → 描画順がそのまま反映されることを確認
- [ ] キー入力で関数を切り替えるUI を実装

---

### 課題3: Z-fighting の再現と対策 
意図的にZ-fightingを再現し、それを解消してください。

**チェックポイント:**
- [ ] 2つの平面を完全に同じ位置に配置してZ-fightingを発生させる
- [ ] near=0.0001, far=10000 で悪化することを確認
- [ ] near=0.1, far=100 に変更して改善を確認
- [ ] 微小オフセット（0.001）で完全に解消
- [ ] 各対策の前後をスクリーンショットで比較
